# User Guide: React, dependencies, and template layout in webview projects

## Core rule

A webview extension has **one** `package.json` at extension root (same level as `src/extension.ts`).
All dependencies (`react`, `antd`, `webpack`, `babel`, etc.) belong there.
`scaffold/bundler/` is **not** an npm package and should not contain `package.json`.

---

## Directory responsibilities

| Path | Responsibility |
|------|------|
| `package.json` at extension root | `dependencies` / `devDependencies` (React, UI libs, bundler/toolchain) |
| `src/webview/` | Product UI code (`registerWebview` entry, panel components) |
| `scaffold/bundler/` | Vendored template artifacts (`webpack.factory.js`, `index.html`, favicon) managed by `vendor:template` |
| `scaffold/webview.webpack.js` | Webpack CLI entry, usually generated by script |

`@easy_vscode/webview` is runtime-only (`registerWebview`, `callVscode`) and is installed in extension root `package.json`.

---

## How to customize React setup

1. Update `react` / `react-dom` versions in root `package.json`, then run `yarn`.
2. Configure syntax/plugins in root `.babelrc` (or `babel.config.js`).
3. Configure TypeScript in root `tsconfig.json`; entry remains `src/webview/index.tsx`.
4. Customize bundling via `scaffold/bundler/webpack.factory.js` or swap the whole `bundler/` directory.

Keep product code in `src/webview/`. Do not place product logic in `scaffold/bundler/`.

---

## Adding Ant Design (or other UI libraries)

1. Install at extension root:

```bash
yarn add antd @ant-design/icons
```

2. Update bundling strategy:
   - Option A: vendor `antd-less` template into `scaffold/bundler/` with `init-webview-template.cjs --vendor --template antd-less --force` or by copying from `easy-vscode/webview-templates/antd-less`.
   - Option B: keep current `webpack.factory.js` and manually merge `less` / `less-loader` rules from `antd-less`.

3. Add root shell component under `src/webview/` (see `AntdWebviewShell.sample.tsx`) and register:

```ts
registerWebview(components, { Root: AntdWebviewShell })
```

Same principle for MUI/Chakra/etc: dependencies stay in root `package.json`; loader/theme handling is done in bundler config and/or webview source.

---

## Why template folders have no `package.json`

This avoids split dependency ownership and version drift.
Dependency ownership is intentionally centralized in extension root `package.json`.
`scaffold/bundler/` is a replaceable build + HTML shell layer.

---

## Project layout summary

- `src/`: business logic and webview UI
- `scaffold/`: generated webpack entry + vendored `bundler/` template

See [scaffold/README.md](./scaffold/README.md).

---

## Script relationship in this demo

- `yarn vendor:template`: copies upstream `minimal-react` template into `scaffold/bundler/` and writes `scaffold/webview.webpack.js`.
- Day-to-day: update root `package.json` and `src/webview/`; rerun vendor only when switching/updating templates (`--force` to overwrite).
