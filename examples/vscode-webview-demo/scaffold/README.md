# scaffold (tooling-managed layer)

Files here are generated/vendor-managed pieces from easy-vscode template flow.
Keep this layer separate from product code under `src/`.

| Path | Meaning |
|------|------|
| `webview.webpack.js` | Webpack CLI entry (`yarn ui-build` uses this). Can be regenerated by `yarn vendor:template` or `init-webview-template`. |
| `bundler/` | Vendored template assets (`webpack.factory.js`, `src/index.html`, etc.) copied from `easy-vscode/webview-templates/*`. Avoid placing product logic here; replace directory when switching templates. |

The extension still has a single dependency entrypoint: root `package.json`.
